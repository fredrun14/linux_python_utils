## Contexte

### Problématique
- `config.name` n'est pas validé → en install `system` (root via sudo), un nom
  comme `../../etc/cron.d/x` permet une écriture **hors périmètre**.
- `installer.py:469-470` : le wrapper est écrit via `write_text` (suit symlink)
  avant le `fchmod O_NOFOLLOW`.
- `checker.py:394-405` : `importlib.metadata` interroge le process courant, pas
  le venv cible → détection de dépendances erronée.
- `install()` fait 116 lignes avec 5 `InstallReport` dupliqués.

### Solution technique retenue
Valider `name` au point d'entrée (`__post_init__`). Écrire le wrapper via
`os.open(O_NOFOLLOW|O_EXCL)`. Restreindre `importlib.metadata` au cas
`venv_path is None`. Factoriser `install()`.

### Fichiers impactés
- `scripts/config.py`, `scripts/installer.py`, `scripts/checker.py`,
  `scripts/paths.py`, `scripts/report.py`, `scripts/__init__.py`
- `tests/test_scripts.py`
