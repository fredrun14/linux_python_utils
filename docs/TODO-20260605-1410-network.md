## Contexte

### Problématique
- `repository.py` : inventaire JSON écrit de façon **non atomique** (corruption
  possible) et lu sans gestion de `JSONDecodeError`.
- `router.py:262` : la validation SSRF accepte les hostnames sans résolution DNS

### Solution technique retenue

éléments dupliqués vers des modules partagés.

### PR1 — ROBUSTESSE (🟠 MAJEUR)

#### `repository.py` — écriture atomique (l.74)
```python
import os
import tempfile

def save(self, devices: list[NetworkDevice]) -> None:
    """Écrit l'inventaire de façon atomique (tmp + os.replace)."""
    data = json.dumps([d.to_dict() for d in devices], ensure_ascii=False)
    fd, tmp = tempfile.mkstemp(dir=str(self._path.parent), suffix=".tmp")
    try:
        with os.fdopen(fd, "w", encoding="utf-8") as f:
            f.write(data)
        os.chmod(tmp, 0o644)
        os.replace(tmp, self._path)  # atomique
    except BaseException:
        os.unlink(tmp)
        raise
```

#### Gestion d'erreurs
| Cas | Condition | Action |
|---|---|---|
| Inventaire corrompu | `JSONDecodeError` | log + liste vide |
| Écriture interrompue | exception pendant save | supprimer le tmp, propager |
| XML nmap invalide | `ParseError` | log + liste vide |
| Hostname hors LAN | résolution publique | `raise`/refus selon option |
