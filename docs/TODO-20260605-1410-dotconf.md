## Contexte

### Problématique
- `applier.py:93` : `content.splitlines()[0]` lève `IndexError` si le contenu est
  vide → crash possible sur une spec mal formée.
- `conf_toml_exporter.py:50` : `read_text` sans gestion de `UnicodeDecodeError`
  (fichiers `/etc` en latin-1).
- `manager.py:82,104,152` : écriture configparser sans `chmod(0o644)` (incohérent
  avec `applier.py` qui force les permissions).
- Duplication : `is_block_present`/`is_block_commented`, pattern configparser, et
  deux regex de section divergentes.

### Solution technique retenue
Gardes défensives + permissions explicites + factorisation des duplications.

### Fichiers impactés
- `dotconf/applier.py`, `conf_toml_exporter.py`, `manager.py`, `line_editor.py`
- `tests/test_dotconf*.py`
