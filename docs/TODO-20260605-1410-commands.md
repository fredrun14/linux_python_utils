## Contexte

### Problématique
Après la migration `subprocess.run` → `Popen`, `runner.py` présente :
- un anti-pattern `type("ProcResult", ...)` (classe anonyme à chaque appel) ;
- un risque de **process orphelin** : sur `TimeoutExpired` dans `run()`, le
  process n'est pas `kill()` avant la remontée (contrairement à `run_streaming`) ;
- un **deadlock possible** en streaming si la commande remplit le buffer stderr ;
- la construction de `CommandResult` dupliquée 6 fois.

### Solution technique retenue
Lire directement `_proc.returncode`/`_stdout`/`_stderr`. Drainer/kill sur timeout.
Centraliser la création de `CommandResult` dans un helper.

### Fichiers impactés
- `src/linux_python_utils/commands/runner.py` — fiabilité + helpers
- `src/linux_python_utils/commands/formatter.py` — dédup Plain/Ansi
- `src/linux_python_utils/commands/base.py` — immutabilité
- `tests/test_commands.py` — timeout/interrupt

## Évolutions à mettre en place (Détail Junior)

### `runner.py` — 🟠 MAJEUR

#### 1. Supprimer le hack `type("ProcResult", ...)` (l.287-294)
Après le `with subprocess.Popen(...)`, lire directement les variables locales
(`_stdout`, `_stderr`, `_proc.returncode`). Le `returncode` reste valide après la
sortie du `with`.

#### 2. Helper de construction du résultat
```python
def _result(
    self,
    command: list[str],
    return_code: int,
    stdout: str,
    stderr: str,
    duration: float,
) -> CommandResult:
    """Construit un CommandResult avec les champs communs."""
    return CommandResult(
        command=command,
        return_code=return_code,
        stdout=stdout,
        stderr=stderr,
        success=return_code == 0,
        duration=duration,
        executed_as_root=self._is_root,
    )
```
Remplacer les 6 `return CommandResult(...)` par `return self._result(...)`.

#### 3. Tuer le process sur timeout (l.268-279)
```python
try:
    _stdout, _stderr = _proc.communicate(timeout=self._timeout)
except subprocess.TimeoutExpired:
    _proc.kill()
    _stdout, _stderr = _proc.communicate()  # draine après kill
    # construire le CommandResult d'erreur via self._result(...)
```

#### 4. Deadlock stderr en streaming (l.394-410)
Choisir UNE option et la documenter :
- option simple : `stderr=subprocess.STDOUT` (fusion) si acceptable ;
- option robuste : drainer stderr dans un thread.
A minima, documenter la limitation dans la docstring de `run_streaming`.

#### Gestion d'erreurs
| Cas | Condition | Action |
|---|---|---|
| Timeout | `TimeoutExpired` | `kill()` + drain + résultat d'erreur |
| Commande introuvable | `FileNotFoundError` | résultat d'erreur |
| Ctrl-C | `KeyboardInterrupt` | `terminate()` + propager (harmoniser run/streaming) |

### `formatter.py` — 🟡 MINEUR (optimisation)
Template Method : remonter `_prefix()` et la construction des textes de base dans
`CommandFormatter`, ne laisser aux sous-classes que `_decorate(text, is_root)`
(Plain = identité, Ansi = couleur). Unifier la visibilité des constantes de
préfixe (privées dans la base).

### `base.py` — 🟡 MINEUR
`command: list[str]` dans une dataclass `frozen=True` reste mutable → utiliser
`tuple[str, ...]` pour une vraie immutabilité. Logs : `shlex.join` au lieu de
`" ".join` pour des commandes fidèles.
